# Contributing

Contributi via Pull Request sono preferiti.

## Linee guida

- Usare file sintetici, anonimi e test fixtures minimali.
- Non committare file reali di clienti, atti o fascicoli.
- Eseguire `pytest` e i controlli di packaging quando rilevanti per la
  modifica proposta.
- Aggiornare `README.md` e la documentazione separata se cambia il
  comportamento del progetto.
- Non modificare workflow, release o packaging senza motivazione
  specifica e verificabile.

## Ambito delle modifiche

Le PR piccole e mirate sono piu' facili da rivedere e da validare prima
della pubblicazione.
